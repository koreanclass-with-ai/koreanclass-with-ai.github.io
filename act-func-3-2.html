설계 근거

중급 학습자가 의견을 말할 때 가장 흔한 문제는 "주장만 하고 근거가 없는 것" 또는 "근거는 있는데 주장과 연결이 안 되는 것"이에요. 주장-근거-예시의 빈칸 프레임을 제공하면, 학습자가 각 요소를 채우면서 자연스럽게 논증 구조를 익히게 돼요.

논증은 "내가 왜 이렇게 생각하는가"를 설명하는 거잖아요. 주장(결론)→근거(이유)→예시(증거)의 3단 구조는 가장 기본적이면서도 효과적인 논증 형식이에요. 이 구조를 시각적으로 보여주고 칸을 채우게 하면, 학습자가 "이 순서로 말하면 된다"는 인식을 갖게 되거든요.

팁: 한 가지 주제로 찬성과 반대 입장을 모두 작성해보면, 같은 주제라도 근거를 달리 선택할 수 있다는 것을 깨닫게 돼요.
설계 과정

프롬프트로 논증 워크시트 만들기

AI에 어떻게 지시하는가?

다음 프롬프트로 TOPIK 4~5급 학습자용 논증 구조 활동을 생성해요.

prompt.txt
# 역할 (Role) 한국어 교육 전문가 # 과제 (Task) TOPIK 4~5급 학습자를 위한 논증 구조 연습 활동 생성 # 조건 (Constraints) - 주제 3개: 외국어 조기 교육, 원격 근무 확대, SNS의 영향 - 각 주제: 주장-근거-예시 프레임(빈칸) + 모범 답안(찬성/반대 각 1개) - 문형: -(으)ㄴ/는다고 생각하다, 왜냐하면 -기 때문이다, 예를 들어, 따라서 # 출력 (Output) JSON 형식
생성된 JSON 구조

JSON 필드 설명
topics →
논증 주제 목록이에요.
frame →
빈칸 프레임이에요. 학습자가 직접 채우는 구조예요.
model_pro →
찬성 측 모범 답안이에요. 주장-근거-예시 완성형이죠.
model_con →
반대 측 모범 답안이에요.
claim →
주장이에요. "~(으)ㄴ/는다고 생각합니다" 문형을 사용해요.
reason →
근거예요. "왜냐하면 ~기 때문입니다" 문형을 사용해요.
example →
예시예요. "예를 들어"로 시작해요.

활동 평가 및 확장

점검: 논증 구조가 유효한가?

이 활동이 학습 목표를 달성했는지 확인하세요.

핵심 체크리스트
  • 프레임이 직관적이고 학습자가 바로 채울 수 있는 구조인가요?
  • 모범 답안의 주장-근거-예시가 논리적으로 연결되나요?
  • 찬성·반대 모범 답안이 모두 설득력 있나요?
난이도 조정 및 확장 방안
조정 유형 설명 활용 시기
난이도 ↓ 3단 → 2단 (주장+근거만) / 예시 없이 주장과 이유만 쓰기 초급 또는 TOPIK 3급 이하
난이도 ↑ 3단 → 5단 (주장+근거+예시+반론+재반론) / 고급 논증 구조 고급 학습자 또는 고등학교 이상
인터랙션 개인 → 짝 피드백 / A가 작성한 논증을 B가 평가 소규모 집단 수업
확장 프레임 완성 → 발표 또는 에세이 / 완성된 프레임으로 2분 발표 수업의 후속 활동
교사 팁: 학습자가 완성한 논증을 발표하게 하면, 같은 주제도 다양한 근거와 예시로 풍부해진다는 것을 깨닫게 돼요. 피드백 때는 "논증이 명확한가", "근거와 예시가 주장을 잘 뒷받침하는가" 두 가지에 집중하세요.

이 활동은 중~고급 학습자가 의견을 체계적으로 표현하도록 돕는 것이 핵심이에요. 주장-근거-예시 프레임을 반복해서 사용하면, 결국 "의견을 말할 때는 이 순서로"라는 습관이 자동화돼요. 학습자의 쓰기와 말하기 능력이 함께 발전하는 경험이 될 거예요.
