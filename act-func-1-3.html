설계 근거

요청의 직접/간접 전략은 실제 대화문을 나란히 비교할 때 가장 잘 드러나요. "창문 좀 닫아 주세요"(직접)와 "여기 좀 춥지 않아요?"(간접)가 같은 상황에서 어떻게 다른 효과를 내는지를, 대화문의 맥락 안에서 보여줄 수 있거든요.

요청은 화자와 청자 사이의 상호작용이에요. 요청 표현만 따로 떼어놓으면 "문장"이 되지만, 대화 속에 놓으면 "전략"이 되잖아요. 상대의 반응(흔쾌히 수락 vs 난색 표시)까지 포함된 대화문은 "이 전략이 효과가 있었는가"를 학습자가 직접 판단하게 해줘요.

AI 프롬프트 설계

프롬프트 구조

# 역할 (Role) 한국어 교육 전문가 (화용론 전공) # 과제 (Task) TOPIK 2~3급 학습자를 위한 요청하기 대화문 + 분석 활동 생성 # 조건 (Constraints) - 대화문 4쌍 (같은 상황의 직접 요청 vs 간접 요청) - 상황: 친구에게 노트 빌리기, 교수에게 추천서, 식당에서 물, 동료에게 교대 부탁 - 각 쌍: 직접/간접의 효과 차이 해설 - 연습 문항 4개: 새 상황에서 적절한 전략 선택 # 출력 (Output) JSON 형식

JSON 출력 형식

{ "pairs": [ { "id": 1, "situation": "친구에게 수업 노트를 빌리고 싶다", "direct": { "dialogue": [ "A: 야, 노트 좀 빌려줘.", "B: 응, 여기." ], "strategy": "직접 요청 (반말, 명령형)" }, "indirect": { "dialogue": [ "A: 혹시 오늘 수업 노트 있어? 내가 빠져서...", "B: 응, 보내줄까?" ], "strategy": "간접 요청 (상황 설명 → 상대가 제안)" }, "analysis": "친구 관계에서는 직접 요청도 자연스럽다. 다만 큰 부탁이면 간접이 더 적절하다." } ] }

JSON 범례

필드 설명
pairs 대화문 쌍 목록이에요.
direct 직접 요청 대화문이에요. 직접적으로 원하는 것을 말해요.
indirect 간접 요청 대화문이에요. 상황 설명이나 질문으로 유도해요.
analysis 직접/간접의 효과 차이를 분석한 해설이에요.

대화문 분석

상황 1: 친구에게 수업 노트 빌리기

✗ 직접 요청
A: 야, 노트 좀 빌려줘.
B: 응, 여기.
전략: 반말, 명령형 · 명확하고 즉각적
✓ 간접 요청
A: 혹시 오늘 수업 노트 있어? 내가 빠져서...
B: 응, 보내줄까?
전략: 상황 설명 + 질문 · 상대가 제안하도록 유도
분석
친구 관계에서는 직접 요청도 자연스럽다. 다만 큰 부탁이면 간접이 더 적절하다. 간접 요청은 상대에게 "너도 할 수 있으면 해"라는 뉘앙스를 주어, 상대가 거절하기 쉽게 만든다.

상황 2: 교수님에게 추천서 부탁하기

✗ 직접 요청
A: 교수님, 추천서 써 주세요.
B: 음... 언제까지 필요해요?
전략: 존댓말, 명령에 가까운 표현 · 부정절이 느껴짐
✓ 간접 요청
A: 교수님, 혹시 시간이 되시면 추천서를 부탁드려도 될까요?
B: 네, 괜찮습니다. 언제까지 필요하세요?
전략: 존댓말 + 완화 장치("-ㄹ 수 있을까요?", "혹시") · 상대의 자유의지 존중
분석
큰 부탁은 간접이 필수다. "혹시", "-ㄹ 수 있을까요?" 같은 완화 장치들이 상대의 거절권을 보장한다. 이런 표현들이 오히려 더 정중하고, 상대에게 긍정적인 인상을 준다.

상황 3: 식당에서 물 부탁하기

✓ 직접 요청
A: 물 한 잔 주세요.
B: 네, 바로 가져오겠습니다.
전략: 존댓말, 명확한 요청 · 종사원 역할이므로 직접이 자연스러움
△ 간접 요청
A: 혹시 물 한 잔 주실 수 있을까요?
B: 네, 바로 가져오겠습니다.
전략: 완화 장치 사용 · 지나친 정중함으로 어색할 수 있음
분석
서비스 상황에서는 직접 요청이 자연스럽고 효율적이다. 간접 요청은 상황에 맞지 않을 수 있다. 대신 존댓말과 "주세요"의 정중함만으로도 충분하다.

상황 4: 동료에게 교대 부탁하기

✗ 직접 요청
A: 내일 교대 좀 해 줘.
B: 어... 힘들 것 같은데...
전략: 반말, 명령형 · 상대에게 거절 여지를 남기지 않음
✓ 간접 요청
A: 내일 교대 가능할까? 급한 일이 생겨서...
B: 내가 한번 봐 볼게. 안 되면 연락할게.
전략: 질문형 + 사유 설명 · 상대의 상황을 배려하는 톤
분석
같은 위치의 동료에게는 큰 부탁일 때 간접 요청이 효과적이다. 상대의 상황을 먼저 고려하는 톤이 "나도 원하지만, 너도 거절할 수 있어"라는 메시지를 전달한다.

♦ 거절 받기

간접 요청의 장점: 거절받을 때 양쪽 모두 "체면"을 지킬 수 있다. "괜찮습니다"라는 거절이 덜 상하다.
