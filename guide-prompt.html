프롬프트 작성 가이드

같은 도구를 써도 프롬프트에 따라 결과가 완전히 달라져요. 역할/과제/조건/출력 네 가지 구조만 잡으면 돼요.

역할 · 과제 · 조건 · 출력 모든 활동에 공통 적용
기본 구조 파트별 상세 자주 하는 실수 응용하는 법
STRUCTURE
프롬프트의 기본 구조

모든 활동 페이지의 프롬프트는 같은 네 파트로 돼 있어요. 이 구조를 알면 어떤 활동이든 프롬프트를 직접 쓸 수 있어요.

역할 (Role)
AI에게 전문가 역할을 부여해요. "한국어 교육 전문가"로 고정하면 대부분의 활동에 통해요.
과제 (Task)
뭘 만들어야 하는지 한 줄로 명시해요. "장소 조사 '에'와 '에서'를 구분하는 빈칸 채우기 문항 생성"처럼요.
조건 (Constraints)
대상 레벨, 문항 수, 어휘 범위, 정답 분포, 제외할 항목 등 구체적인 제약을 거세요. 조건이 많을수록 결과가 좋아요.
출력 (Output)
결과물의 형식을 지정해요. 웹앱에 넣을 거면 JSON, 인쇄용이면 "표 형식으로 정리해줘".
기본 틀
역할: 한국어 교육 전문가 과제: [만들 것을 한 줄로] 조건: - 대상: [TOPIK 몇 급] - 문항 수: [몇 개] - [추가 제약 조건들] 출력: [JSON / 표 / 대화문 형식 등]
DETAIL
파트별 상세 가이드
역할 — 보통 바꿀 필요 없어요

"한국어 교육 전문가"로 고정하면 문법 문항, 대화문, 읽기 텍스트, 역할극 카드 등 대부분의 활동에 적용돼요.

과제 — 핵심은 구체적으로
❌ "문법 문제 만들어 줘" — 너무 모호해요
✅ "장소 조사 '에'와 '에서'를 구분하는 빈칸 채우기 문항 생성" — 정확해요
조건 — 많을수록 좋아요

빠지기 쉬운 조건들을 정리했어요:

정답 분포 — "'에' 5개, '에서' 5개 균형 배치"가 없으면 한쪽으로 쏠려요.

제외 항목 — "이동의 '에'(가다/오다)는 제외"처럼, AI가 섞어 넣기 쉬운 걸 명시적으로 빼세요.

어휘 범위 — "초급 기본 동사만 사용"이 없으면 중급 어휘가 슬쩍 들어와요.

해설 포함 여부 — "각 문항에 정답 해설, 오답 해설 포함"을 안 쓰면 해설이 빠져요.

출력 — 용도에 따라 바꾸세요

웹앱에 넣을 거면 → "JSON 형식으로" — 코드 생성 도구에 바로 붙여넣을 수 있어요.

인쇄용이면 → "A4 워크시트 형태로 정리해줘" — 바로 프린트 가능.

교사용 참고자료면 → "표로 정리해줘" — 한눈에 보기 좋아요.

MISTAKES
자주 하는 실수
❌ "문제 좀 만들어 줘" — 수준도 들쭉날쭉, 같은 유형만 반복, 해설도 빠져요
❌ 조건 없이 과제만 던지기 — AI가 알아서 잘 하겠지? 안 돼요
❌ 출력 형식 안 쓰기 — 결과가 문단으로 나와서 정리하느라 시간 더 걸려요
❌ 해설 기준이 문항마다 달라지게 두기 — "이건 장소니까", "이건 행동이니까" 식으로 기준이 바뀌면 학습자가 규칙을 귀납할 수 없어요
오답 해설은 모든 문항이 동일한 판단 기준을 따라야 해요. 문항마다 기준이 바뀌면 학습자가 혼란스러워요.
ADAPT
다른 활동에 응용하는 법

역할은 그대로 두고, 과제와 조건만 바꾸면 어떤 활동이든 만들 수 있어요.

역할
그대로 두세요. "한국어 교육 전문가"는 모든 활동에 통해요.
과제
목표만 바꾸세요. "이/가 vs 은/는 구분 문항 생성", "카페 주문 상황 역할극 카드 생성", "찬반 의견 카드 세트 생성"
조건
활동 특성에 맞게 조정하세요. 말하기면 "대화 6턴 이내", 읽기면 "200자 내외", 과제형이면 "조건 카드 3장 + 결정 시트 1장"
출력
앱이면 JSON, 인쇄면 표, 카드형이면 "카드 1장당 하나의 JSON 객체로"
각 활동 페이지에 해당 활동에 맞는 프롬프트가 이미 들어있어요. 처음에는 그걸 그대로 복사해서 쓰고, 익숙해지면 여기 원칙을 참고해서 직접 수정하세요.
