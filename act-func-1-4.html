설계 근거

위로 상황에서 가장 중요한 건 "먼저 상대의 감정을 인정하는 것"이에요. 대화문에서 상대의 고민을 먼저 읽은 후 빈칸에 위로 표현을 채우는 형식은, 학습자가 "상대가 무엇 때문에 힘들어하는가"를 파악한 뒤 반응하는 순서를 자연스럽게 연습시켜 줘요.

위로는 상대의 발화에 대한 반응이잖아요. 상대가 무슨 말을 했는지에 따라 적절한 위로가 달라지거든요. 대화문 완성은 상대의 발화를 먼저 제시하고 학습자의 반응을 요구해서, "듣고 → 공감하고 → 반응하는" 위로의 순서를 구조적으로 연습시켜요.

프롬프트 & JSON 구조

프롬프트

# 역할 (Role)
한국어 교육 전문가

# 과제 (Task)
TOPIK 3급 학습자를 위한 위로·격려 대화 완성 활동 생성

# 조건 (Constraints)
- 시나리오 5개: 시험 실패, 직장 스트레스, 이별, 건강 문제, 취업 실패
- 각 시나리오: A의 고민 발화(2~3턴) + B의 빈칸(학습자가 채움)
- 모범 답안 3개씩 (공감, 격려, 조언)
- 부적절한 답안 1개씩 (비교용)

# 출력 (Output)
JSON 형식

JSON 구조 & 설명

{
  "scenarios": [
    {
      "id": 1,
      "topic": "시험 실패",
      "dialogue_A": [
        "시험을 못 봤어요.",
        "많이 공부했는데... 너무 속상해요."
      ],
      "blank_position": "B의 첫 반응",
      "good_answers": [
        {
          "text": "많이 속상하겠다. 열심히 했는데...",
          "strategy": "공감"
        },
        {
          "text": "다음에 더 잘할 수 있을 거야. 힘내!",
          "strategy": "격려"
        },
        {
          "text": "뭐가 제일 어려웠어? 같이 공부할까?",
          "strategy": "구체적 도움 제안"
        }
      ],
      "bad_answer": {
        "text": "그래? 나는 잘 봤는데.",
        "reason": "상대의 감정을 무시하고 자신의 경험으로 전환하여 부적절하다."
      }
    }
  ]
}

JSON 필드 설명

  • scenarios: 위로 시나리오 목록이에요.
  • dialogue_A: A의 고민 발화예요. 2~3턴으로 구성돼요.
  • good_answers: 모범 답안 3개예요. 공감, 격려, 조언 전략이 있어요.
  • bad_answer: 부적절한 답안이에요. 왜 부적절한지도 함께 설명해요.
  • strategy: 위로 전략 유형이에요. 학습자가 다양한 반응을 배울 수 있어요.

연습 활동: 부적절한 위로 고르기

위로 표현 선택 활동
TOPIK 3급 · 대인 관계 · 3문항
부적절한 위로를 고르세요. 상대의 감정을 무시하거나 공감하지 못한 답을 찾아요.

Q1. 시험 실패

A: "시험을 못 봤어요. 많이 공부했는데... 너무 속상해요."

Q2. 직장 스트레스

A: "요즘 회사에서 일이 너무 많아요. 매일 야근이에요."

Q3. 취업 실패

A: "취업 시험에 또 떨어졌어요. 이번이 세 번째예요."
