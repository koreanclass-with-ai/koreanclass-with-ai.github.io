설계 근거

사과는 단순히 "미안합니다"를 말하는 게 아니에요. 상황의 심각도에 따라 사과의 강도와 전략이 달라져야 하거든요. 시나리오는 "무엇이 잘못되었는가"와 "상대방이 누구인가"를 동시에 제시해서, 학습자가 적절한 사과 전략을 선택하도록 유도해요.

사과에서 핵심은 "얼마나 심각한 잘못인가"에 따라 사과의 구성 요소(인정, 설명, 보상 제안, 재발 방지 약속)를 조합하는 거예요. 작은 실수에 과도한 사과를 하면 어색하고, 큰 잘못에 가벼운 사과를 하면 부적절하잖아요. 시나리오를 심각도 순으로 배치하면 이 "조절"을 체계적으로 연습할 수 있어요.

제작 과정

Step 1: 프롬프트 작성

# 역할 (Role)
한국어 교육 전문가 (화용론 전공)
 
# 과제 (Task)
TOPIK 3급 학습자를 위한 사과하기 시나리오 생성
 
# 조건 (Constraints)
- 시나리오 6개, 심각도 3단계 × 관계 2종류
경미+친구, 경미+직장, 중간+친구, 중간+직장, 심각+친구, 심각+직장
- 각 시나리오: 상황 설명, 적절한 사과 전략 구성 요소, 모범 사과문, 부적절한 사과문(비교용)
 
# 출력 (Output)
JSON 형식
📌 주의: "부적절한 사과문"을 포함시키는 이유는, 학습자가 "과도한 사과도 부적절할 수 있다"는 걸 인식하게 하기 위해서예요. 한국어 학습자는 "정중할수록 좋다"고 생각하는 경향이 있는데, 상황에 비해 과도한 사과는 오히려 관계를 불편하게 만들거든요.

Step 2: JSON 응답 예시

{
  "scenarios": [
    {
      "id": 1,
      "severity": "경미",
      "relationship": "친구",
      "situation": "카페에서 만나기로 했는데 5분 늦었습니다.",
      "strategy": ["사과 표현", "간단한 이유"],
      "good_example": "미안, 버스가 늦게 와서. 오래 기다렸어?",
      "bad_example": "정말 죄송합니다. 다시는 이런 일이 없도록 하겠습니다.",
      "bad_reason": "친구에게 5분 늦은 것에 과도하게 격식적인 사과는 어색하다."
    },
    { "id": 2, "...": "..." }
  ]
}

JSON 항목 설명

scenarios: 사과 시나리오 목록이에요.

severity: 심각도 단계예요. 경미/중간/심각 3단계가 있어요.

relationship: 화자 관계예요. 친구와 직장으로 나뉘어요.

strategy: 적절한 사과 전략 구성 요소예요.

good_example: 모범 사과문이에요. 상황에 맞는 적절한 사과 표현이죠.

bad_example: 부적절한 사과문이에요. 왜 부적절한지 비교용으로 들어가요.

bad_reason: 부적절한 사과문이 왜 상황에 맞지 않는지 설명해요.

연습

아래 시나리오를 읽고, 상황에 맞는 사과 표현을 선택해보세요. 심각도와 관계에 맞는 적절한 사과가 무엇인지 판단하는 연습이에요.

❶ 연습 1: 경미한 상황 + 친구
어떤 사과 표현이 더 자연스러울까요?
시나리오
카페에서 만나기로 했는데 5분 늦었습니다. 친구와의 약속이에요.
✓ 미안, 버스가 늦게 와서. 오래 기다렸어?
✗ 정말 죄송합니다. 다시는 이런 일이 없도록 하겠습니다.
설명: 친구에게 5분 늦은 것은 작은 실수예요. 첫 번째 표현처럼 "미안"이라는 간단한 사과와 이유를 함께 말하는 게 자연스러워요. 두 번째 표현은 너무 격식적이고 과도해서 오히려 어색할 수 있어요.
❷ 연습 2: 중간 정도의 상황 + 직장
어떤 사과 표현이 더 적절할까요?
시나리오
중요한 회의에 교통 체증으로 30분을 지각했습니다. 직장 상사와의 회의예요.
✗ 아, 좀 늦었네요.
✓ 죄송합니다. 교통 사고가 있어서 늦었습니다. 다시는 이런 일이 없도록 하겠습니다.
설명: 직장의 중요한 회의에 30분을 지각한 것은 사소한 일이 아니에요. 따라서 첫 번째처럼 "아, 좀 늦었네요"라고 가볍게 넘기면 상대방에게 불성실해 보여요. 두 번째처럼 "죄송합니다" + 구체적인 이유 + 재발 방지 약속을 포함해서 진지한 사과를 해야 해요.
❸ 연습 3: 심각한 상황 + 친구
어떤 사과 표현이 더 적절할까요?
시나리오
친구와 약속한 날 전혀 연락 없이 약속을 까먹었습니다. 친구에게 정말 미안한 상황이에요.
✗ 아 미안, 깜빡했어.
✓ 정말 미안해. 내가 완전히 잊어버렸어. 진짜 할 말이 없다. 다음에 내가 꼭 사줄게.
설명: 중요한 약속을 완전히 깜빡한 것은 심각한 실수예요. 비록 친구 사이지만 첫 번째처럼 "아 미안, 깜빡했어"라고 너무 가볍게 사과하면 상대 친구가 상할 거예요. 두 번째처럼 "정말 미안해" (강한 사과) + 진지한 표현 + 보상 제안 (내가 사줄게)을 함께 하는 게 자연스러워요.

체크리스트

시나리오 검증

  • 6개 시나리오의 심각도가 순차적으로 올라가나요? (경미 → 중간 → 심각)
  • 모범 사과문이 화자 관계(친구 vs 직장)에 맞는 높임법을 사용하나요?
  • 부적절한 사과문이 "왜 부적절한지"가 명확하게 설명 가능한가요?

변형 및 확장

변형 유형 설명
기능 교체 사과 → 감사 표현 / 심각도 대신 "감사의 정도"로 배열
난이도 ↑ 모범 사과문 없이 학습자가 직접 작성 / 전략 구성 요소만 체크
문화 비교 한국식 사과 vs 학습자 모국의 사과 / "우리나라에서는 어떻게 사과하나?" 토론

추가 학습 자료

🎯 심화 포인트: 사과의 각 구성 요소(인정, 설명, 보상, 재발 방지)가 심각도에 따라 어떻게 조합되는지 명시적으로 지도하면 학습자의 이해도가 높아져요.

💬 토론 활동: 각 시나리오마다 "당신이라면 어떻게 사과했을까?"라고 물어본 다음, 다양한 답변들을 모아서 "왜 이 사과가 더 나을까?"를 비교 분석하는 활동도 좋아요.
