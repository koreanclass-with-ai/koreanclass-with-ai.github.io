설계 근거

비교 표현은 비교 대상이 있어야 사용할 동기가 생기잖아요. "서울은 부산보다 크다"라는 문장을 만들려면 서울과 부산 두 대상의 정보가 필요하거든요. 비교 자료(표, 그래프, 정보 카드)를 제공하면 학습자가 자연스럽게 비교 표현을 사용하게 돼요.

비교·대조는 정보를 조직하는 가장 기본적인 방식 중 하나예요. "A와 B의 공통점은 무엇이고, 차이점은 무엇인가"를 설명하는 과정에서 학습자는 정보를 구조화하는 능력과 그것을 언어로 표현하는 능력을 동시에 연습하게 되죠.

프롬프트로 자료 생성하기

# 역할 (Role)
한국어 교육 전문가

# 과제 (Task)
TOPIK 3~4급 학습자를 위한 비교·대조 설명 활동 생성

# 조건 (Constraints)
- 주제 3개: 서울 vs 부산, 한국 학교 vs 학습자 나라 학교, 한국 음식 vs 서양 음식
- 각 주제: 비교 정보 표, 공통점 2개, 차이점 3개
- 목표 표현: -보다, -만큼, 반면에, -는 데 비해, 공통점/차이점, 비슷하다/다르다
- 모범 비교문 3~4문장

# 출력 (Output)
JSON 형식 
{
  "topics": [
    {
      "id": 1,
      "subject_A": "서울",
      "subject_B": "부산",
      "comparison_data": {
        "인구": {
          "A": "약 950만 명",
          "B": "약 340만 명"
        },
        "특징": {
          "A": "수도, 정치·경제 중심",
          "B": "항구도시, 관광 중심"
        }
      },
      "similarities": [
        "한국의 대도시",
        "지하철이 있다"
      ],
      "differences": [
        "인구 차이",
        "기후 차이"
      ],
      "model_paragraph": "서울은 부산보다 인구가 많다..."
    }
  ]
}

출력 요소 설명

topics → 비교 주제 목록이에요.
subject_A / subject_B → 비교 대상 A와 B예요.
comparison_data → 비교 정보 표예요. 항목별 데이터가 들어가요.
similarities → 공통점이에요.
differences → 차이점이에요.
model_paragraph → 모범 비교문이에요. 목표 표현이 사용돼요.
