섞여 있는 어절 카드를 순서대로 놓아 문장을 완성하는 앱이에요. 프롬프트 하나로 문장을 뽑고, 자동채점 앱까지 만드는 과정을 단계별로 안내해요.

"한국어는 동사가 마지막"이라는 걸 머리로는 알아도, 막상 문장을 만들면 모어의 어순이 튀어나와요. 영어권 학습자가 "저는 만났어요 친구를"처럼 말하는 게 대표적이죠.

배열형으로 만든 이유가 있어요. 초급 학습자에게 문장을 처음부터 쓰게 하면 어휘·철자·어순을 동시에 감당해야 해요. 어절 카드를 주면 어휘와 철자는 해결된 상태라, "어떤 순서로 놓을까"라는 어순 판단에만 집중할 수 있어요.

카드를 직접 움직이는 것도 포인트예요. 문장 구조를 눈과 손으로 만지는 경험이라서, 설명을 듣는 것보다 어순 감각이 빨리 붙어요. 조사가 어절에 붙어 있으니 "'친구를'은 목적어니까 동사 앞" 같은 단서 찾기 연습도 자연스럽게 돼요.

BUILD
🛠 따라 만들기 — 4단계

요즘은 Claude나 ChatGPT 같은 대화형 AI에 바로 부탁하면, 작동하는 웹앱까지 한 번에 만들어줘요. 코드를 몰라도 괜찮아요. 아래 4단계만 따라오세요.

1
무엇을 만들지 정하기
프롬프트를 쓰기 전에 5초만 생각해요

만들기 전에 딱 세 가지만 정하면 돼요. 이게 곧 프롬프트의 재료가 돼요.

무엇을
한국어 기본 어순 (주어 → 때·장소 → 목적어 → 서술어) — 가르칠 목표예요.
어떻게
어절 카드 배열 — 카드를 눌러(또는 끌어) 순서대로 놓는 방식이에요.
얼마나
8문항, 한 문장은 3~4어절 — 초급은 5어절을 넘기지 않는 게 좋아요.
2
프롬프트로 한 번에 만들기
아래 프롬프트를 복사해서 그대로 붙여넣으세요

Claude나 ChatGPT를 열고 아래 프롬프트를 붙여넣으면, 문장부터 작동하는 앱까지 한 번에 만들어줘요.

📐 프롬프트를 잘 쓰는 법이 궁금하다면 → 프롬프트 작성 가이드 보기
⚠️ "어순 배열 문제 만들어줘"라고만 하면, AI가 두 가지 실수를 자주 해요. ① 조사를 단어에서 떼어 '친구' '를'처럼 따로 카드로 만들고, ② 다른 어순도 자연스러운 문장을 내요("어제 저는 친구를 만났어요"도 맞는데 오답 처리). 그래서 프롬프트에 "조사는 앞말에 붙여 한 어절로"와 "정답 어순이 하나뿐인 문장만"을 꼭 넣으세요.
기본 프롬프트
# 만들 것 한국어 학습용 '어절 순서 배열' 학생용 웹앱 # 내용 - 대상: 초급 학습자 - 문항: 8개, 한 문장은 3~4어절 - 조사는 앞말에 붙여 한 어절로 (예: "친구를" — "친구"+"를" 금지) // 주어로 시작하고, 정답 어순이 하나뿐인 문장만 사용 - 어휘: 초급 기본 어휘 (학교, 친구, 먹다, 공부하다 등) - 각 문항마다 순서 규칙 해설 - 해설은 문법 용어 없이 쉬운 말로 (예: "'무엇을'은 '뭐 해요' 앞에 와요") // 주어·목적어·서술어 같은 용어는 학생 화면에 쓰지 않기 # 동작 - 섞인 어절 카드를 누르면 순서대로 윗줄에 들어가기 - 놓은 카드를 다시 누르면 아래로 되돌아오기 - 다 놓으면 [확인] 버튼 → 정답/오답과 해설 보여주기 - 8문항을 다 풀면 점수 보여주기 - 휴대폰에서는 드래그 대신 터치로 조작되게 # 중요 - 학생 화면에는 문제·카드·피드백·점수만 보이게 // 정답 어순 목록, 제작 설명은 학생에게 보이지 않게
💡 위 프롬프트의 뼈대(만들 것·내용·동작·중요)는 그대로 두고, 목표만 바꾸면 어떤 활동이든 만들 수 있어요. 색이나 카드 모양 같은 디자인은 굳이 적지 않아도 알아서 만들어줘요.
🔁 다른 활동으로 바꾸려면

전체를 다시 쓰지 말고, 아래 세 곳만 바꾸세요.

단위 바꾸기
어절 → 문장 카드로: "대화 4~6턴을 순서대로 배열"이라고 하면 대화 순서 배열이 돼요. 요리법이면 절차 배열.
난이도 바꾸기
"5~6어절로", "안 쓰이는 교란 어절 1개 섞기"를 추가하면 중급용이 돼요.
초점 바꾸기
"한국어·영어 어순을 나란히 비교하게"라고 하면 어순 비교 활동이 돼요.
3
나온 앱 점검하고 손보기
바로 학생에게 주지 말고, 먼저 점검하고 고쳐요

프롬프트를 넣으면 앱이 바로 나와요. 하지만 한 번에 완벽하진 않아요. 먼저 무엇을 고칠지 찾고, 그 다음 고치면 돼요.

먼저, 무엇을 고칠지 점검하기

코드를 읽을 필요 없어요. 학생처럼 직접 풀어보며 아래만 확인하세요.

미리보기에서 확인할 것
  • 첫 화면에 섞인 카드와 놓을 자리가 바로 보이나요?
  • 카드를 잘못 놓았을 때 다시 뺄 수 있나요?
  • 조사가 앞말에 붙어 있나요? ("친구를"이 한 카드인가요?)
  • 다른 어순으로도 자연스러운 문장이 오답 처리되지 않나요?
  • 학생 화면에 정답 어순 목록·교사용 설명이 안 보이나요?
  • 스마트폰 크기에서도 카드를 누르기 편한가요?
⚠️ AI는 빠르게 초안을 만드는 조수일 뿐, 교육적 판단은 선생님 몫이에요. 특히 어순 문제는 "이 순서만 맞는가?"를 문항마다 소리 내어 읽으며 확인하세요.
고치는 방법은 두 가지예요

고칠 곳을 찾았으면, 아래 두 방법 중 편한 쪽으로 수정하세요. 대부분은 방법 ①로 충분해요.

방법 ①
말로 고치기 제일 쉬움

고치고 싶은 걸 그냥 말하면 돼요. 코드를 볼 필요도 없어요. 대부분 이걸로 해결돼요.

이렇게 말하면 돼요
2번 문장은 '어제'가 앞에 와도 자연스러워. 어순이 하나로 고정되는 문장으로 바꿔줘. 조사가 따로 카드로 나왔어. 앞말에 붙여서 한 카드로 만들어줘. 놓은 카드를 다시 뺄 수가 없어. 누르면 되돌아오게 해줘. 카드가 작아서 휴대폰에서 누르기 힘들어. 크게 해줘. 맞혔을 때 완성된 문장을 한 번 더 크게 보여줘.
방법 ②
코드로 고치기 더 정밀하게

여러 곳을 한꺼번에 손보거나 세밀하게 조정하고 싶으면, AI한테 "코드를 보여줘"라고 해서 직접 고칠 수 있어요. AI가 만든 문항은 코드 안에 이렇게 들어 있어요. 파란 글씨에 마우스를 올려보세요.

문항 데이터 (일부)
{ answer: ["저는", "어제", "친구를", "만났어요"], explain: "'언제'(어제)는 '누가' 다음에, '무엇을'은 '뭐 해요' 앞에 와요." }, { answer: ["동생이", "빵을", "먹어요"], explain: "'누가' → '무엇을' → '뭐 해요' 순서예요." }
💡 answer에 적은 순서가 정답 어순이에요 — 카드는 여기서 자동으로 섞여 나와요. 이 부분을 고치면 문항이 바뀌어요.
📄 AI가 만든 전체 코드를 구역별 설명과 함께 보기 →
고치고 → 다시 확인하고를 마음에 들 때까지 반복하면 돼요. 급하면 방법 ①만으로도 충분해요.
4
학생에게 내보내기
링크나 QR코드로 공유해요

앱이 완성되면 링크 하나로 학생에게 줄 수 있어요. 학생은 각자 휴대폰으로 접속해서 자기 속도로 풀면 돼요. 인쇄도 복사도 필요 없어요.

🔍 [심화] 문장을 더 꼼꼼히 검토하고 싶다면 — 2단계로 나눠 만들기▶

한 번에 앱까지 만들면 빠르지만, 문장을 하나하나 검토하긴 어려워요. 어순 문제는 특히 "다른 순서도 되는 문장"이 섞이기 쉬워서, 문장의 질을 통제하고 싶으면 두 단계로 나눠서 만드세요.

1단계 — 문장만
// 앱 말고 문장 목록만 먼저 받기 위 조건으로 문장 8개를 표로만 만들어줘. 각 문장에 어절 분리, 정답 어순, 해설을 넣고, 다른 어순도 자연스러운 문장이 있으면 표시해줘.

받은 문장을 소리 내어 읽으며 검토하고 고친 뒤, "이 데이터로 웹앱 만들어줘"라고 이어서 말하면 돼요.
