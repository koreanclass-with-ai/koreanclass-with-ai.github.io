언어 기능읽기
규모9쌍 / 7–10분
배포 형태웹 기반 또는 인쇄용 카드

빈칸 채우기에서는 한 문장 안에서 하나의 문법만 판단하거든요. 하지만 매칭에서는 여러 표현과 여러 상황을 한눈에 놓고 대비할 수 있어요.

유사 문법(-아서/-(으)니까/-기 때문에)은 모두 "이유"를 나타내지만 사용 맥락이 다르잖아요. 학습자가 이 차이를 인식하려면 "이 표현이 자연스러운 상황"과 "부자연스러운 상황"을 동시에 경험해야 해요.

그래서 매칭이 유사 문법 변별을 가르치기에 가장 적합한 형식이에요. 여러 선택지를 한눈에 보면서 "아, -아서는 이럴 때, -(으)니까는 저럴 때 쓰는구나"라는 차이를 자연스럽게 느낄 수 있거든요.

BUILD
🤖 Step 1. AI한테 재료를 만들어 달라고 하세요

Claude, ChatGPT, Gemini 아무거나 열고 아래 프롬프트를 그대로 붙여넣으세요.

📐 프롬프트를 잘 쓰는 법이 궁금하다면 → 프롬프트 작성 가이드 보기
⚠️ 이 활동에서 특히 주의하세요
핵심은 "이 상황에서 다른 두 문법이 왜 부적절한가"를 AI가 제대로 구분하느냐예요. 출력 결과를 받으면, 각 상황 문장에 세 표현을 모두 넣어 보고 자연스러운지 확인해야 해요. AI가 생성한 상황이 실제로는 두 문법 모두에 어울리는 경우가 종종 있거든요.
prompt.txt
# 역할 (Role) 한국어 교육 전문가 # 과제 (Task) TOPIK 3급 학습자를 위한 문법-상황 매칭 문항 생성 # 조건 (Constraints) - 대상 문법: 이유 표현 3가지 (-아서, -(으)니까, -기 때문에) - 각 문법에 대해 3개의 상황 문장 (총 9쌍) - 상황 문장은 해당 문법만 자연스럽고, 다른 두 문법은 부적절하도록 설계 - 각 쌍에 대해 왜 이 문법이 적합한지 1줄 해설 포함 # 출력 (Output) JSON 형식
역할과 출력 형식은 그대로 두고, 과제와 조건만 바꾸면 어떤 문법 항목이든 만들 수 있어요.
🔁 이렇게 바꿔서 응용하세요

문법 세트를 바꾸거나, 난이도를 조정해서 응용할 수 있어요.

역할
그대로 두세요. 교육 전문가 역할은 어떤 활동에도 통해요.
과제
문법 세트를 바꾸세요. "-는데 vs -(으)ㄴ데 vs -더니", "-려고 vs -(으)러 vs -기 위해" 등으로요.
조건
난이도를 조정하세요. 매칭 대신 선택형으로 바꾸면 쉬워지고, 상황 문장을 대화문으로 확장하면 어려워져요.
출력
앱으로 만들 거면 JSON, 인쇄용이면 "카드 형식으로 정리해줘"로 바꾸세요.
📤 Step 2. 결과를 확인하세요

AI가 아래처럼 JSON 형식으로 문항을 만들어 줘요. 직접 수정해서 쓸 수도 있어요. 파란색 키에 마우스를 올려보세요.

output.json
{ "pairs": [ { "id": 1, "grammar": "-아서", "situation": "시간이 없어서 택시를 탔어요.", "explanation": "-아서는 원인-결과가 시간순으로 이어질 때 자연스럽다." }, { "id": 2, "grammar": "-(으)니까", "situation": "비가 오니까 우산을 가져가세요.", "explanation": "-(으)니까는 명령·청유문 앞에 올 수 있다." }, { "id": 3, "grammar": "-기 때문에", "situation": "교통사고가 나기 때문에 도로가 막힙니다.", "explanation": "-기 때문에는 격식적/객관적 이유를 나타낼 때 쓴다." }, { "id": 4, "...": "..." } ] }
grammar
→문법 표현이에요. -아서, -(으)니까, -기 때문에 중 하나.
situation
→해당 문법이 자연스러운 상황 문장이에요.
explanation
→왜 이 문법이 적합한지 1줄 해설이에요.
⚠️ AI가 만든 문항은 꼭 확인하세요. 생성된 상황에 모든 문법을 대입해서 실제로 자연스러운지 검증하는 것이 교사의 책임이에요.
