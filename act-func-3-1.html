설계 근거

의견 표현 활동에서 가장 어려운 건 "무엇에 대해 의견을 말할 것인가"예요. 카드에 논제와 함께 찬성/반대 측의 핵심 근거 키워드를 제시하면, 학습자가 "의견은 있는데 한국어로 못 말하겠다"는 상태에서 출발할 수 있거든요. 근거가 전혀 없이 의견만 요구하면 언어 이전에 사고의 부담이 너무 커요.

찬반은 가장 단순한 의견 구조잖아요. 찬성 아니면 반대, 둘 중 하나를 고르면 되니까요. 이 이분 구조 안에서 "왜 찬성인가/왜 반대인가"를 설명하는 것이 논증의 기본 단위예요. 더 복잡한 논증은 이 기본 단위를 먼저 익힌 뒤에 가능하거든요.

프롬프트 구성

AI 프롬프트로 카드 만들기

아래 프롬프트를 복사해 Chat GPT, Claude, Gemini 등에 붙여 넣으세요.

prompt-debate-card.md
# 역할 (Role) 한국어 교육 전문가 # 과제 (Task) TOPIK 4~5급 학습자를 위한 찬반 의견 카드 생성 # 조건 (Constraints) - 논제 4개: 대학 출석 의무화, 동물원 폐지, 초등학생 스마트폰 사용 금지, AI 대체 가능 직업의 미래 - 각 논제: 찬성 키워드 3개 + 반대 키워드 3개 + 논제 배경 설명 - 활동 순서: 입장 선택(1분) → 근거 정리(3분) → 발표(2분) - 발표 문형: -(으)ㄴ/는다고 생각하다, 왜냐하면 -기 때문이다, 예를 들어, 따라서 # 출력 (Output) JSON 형식
프롬프트 출력 (JSON)

AI의 응답은 다음 형식을 따릅니다. 키를 클릭하면 설명을 볼 수 있어요.

output.json
{ "topics": [ { "id": 1, "topic": "대학교에서 출석을 의무로 해야 하는가?", "background": "많은 대학교에서 출석을 성적에 반영하고 있다. 그러나 최근 온라인 강의가 늘면서 출석 의무화에 대한 논쟁이 있다.", "pro_keywords": ["학습 효과 향상", "성실한 태도 형성", "수업 참여 동기"], "con_keywords": ["학생 자율성 침해", "형식적 출석", "온라인 대체 가능"], "useful_patterns": ["-(으)ㄴ/는다고 생각합니다", "왜냐하면 -기 때문입니다", "예를 들어", "따라서"] }, { "id": 2, "..." } ] }
topics → 찬반 논제 목록이에요.
topic → 논제(토론 주제)예요. 질문 형태로 제시돼요.
background → 논제 배경 설명이에요. 왜 이 주제가 논쟁이 되는지 설명해요.
pro_keywords → 찬성 측 핵심 근거 키워드예요.
con_keywords → 반대 측 핵심 근거 키워드예요.
useful_patterns → 발표에 사용할 문형 목록이에요.
프롬프트 조정 방법 (Application Layers)
역할

그대로 두세요. 교육 전문가로서의 관점이 일관성 있는 카드를 만들어요.

과제

논제를 교체하면 다양한 주제로 만들 수 있어요. 예: "한국의 주 4일 근무제 도입", "대도시 자동차 진입 제한"

조건

키워드 수를 늘리거나 줄여서 난이도를 조절할 수 있어요. 예: "각 3개 → 5개" 또는 "각 3개 → 2개"

출력

앱이면 JSON, 인쇄용이면 "A4 카드 형식"으로 변경. 또는 마크다운 표 형식도 가능해요.

활동 예시

카드 기반 활동 데모

이 활동은 웹앱 데모 대신 완성된 자료의 모습을 보여줍니다.

논제 1: 대학교에서 출석을 의무로 해야 하는가?

많은 대학교에서 출석을 성적에 반영하고 있다. 그러나 최근 온라인 강의가 늘면서 출석 의무화에 대한 논쟁이 있다.
찬성
  • 학습 효과 향상
  • 성실한 태도 형성
  • 수업 참여 동기
반대
  • 학생 자율성 침해
  • 형식적 출석
  • 온라인 대체 가능

  • 예를 들어
  • 따라서

활동 진행 방법
교사 가이드
  • 카드를 화면에 띄우거나 인쇄해서 나눠주세요.
  • 1분: 학습자가 찬성 또는 반대 중 하나를 고르게 해요.
  • 3분: 선택한 쪽의 키워드를 이용해 자기 의견을 1~2문장으로 정리하게 해요.
  • 2분: 준비된 의견을 발표하거나 짝과 나누게 해요.
점검·확장

카드 만들기 체크리스트

AI가 생성한 카드가 수업에 맞는지 점검하세요.

  • 논제가 학습자에게 친숙하고 의견이 나뉠 수 있는 주제인가요?
  • 찬성과 반대 키워드가 균형 있고 설득력 있나요?
  • 배경 설명이 논제를 이해하는 데 충분한가요?
난이도·활동 확장
변형 설명
난이도 ↓
(쉽게)		 • 키워드 → 완성 문장으로 제공
• 근거를 직접 만드는 대신 주어진 근거 중 선택
• 문형 예시 추가
난이도 ↑
(어렵게)		 • 키워드 없이, 논제만 제공
• 학습자가 직접 근거 발굴
• 3개 이상의 입장(찬성·반대·중도)으로 확장
상호 작용 • 개인 발표 → 짝 토론
• 찬성 측과 반대 측이 3분간 토론
• Q&A 라운드 추가
활동 확장 • 발표 → 에세이 작성
