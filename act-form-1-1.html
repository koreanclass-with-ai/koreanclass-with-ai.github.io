'에'와 '에서'는 초급에서 가장 많이 헷갈리는 조사 중 하나예요. 설명은 이해했는데 막상 문장에서 고르려면 헷갈리는 거, 선생님들도 잘 아시죠?

선택형으로 만든 이유가 있어요. 초급 학습자는 한글 입력 자체가 부담이에요. 선택지를 두 개로 줄이면 "에/에서 중 뭐가 맞지?"라는 판단에만 집중할 수 있어요.

문항마다 해설이 다르게 나오는 것도 포인트예요. "'공부하다'는 무언가를 하는 말이에요. 하는 곳에는 '에서'를 써요." 이런 식으로요 — 학생 화면이니 문법 용어 대신 쉬운 말로요. 이게 자동채점 웹앱의 가장 큰 장점이에요.

📐 프롬프트를 잘 쓰는 법이 궁금하다면 → 프롬프트 작성 가이드 보기
⚠️ '에'와 '에서'를 구분하는 문항을 만들어달라고만 하면, AI가 두 가지 실수를 자주 해요. ① 이동의 '에'(학교에 가요)를 섞고, ② 정답표·해설을 학생 화면에 그대로 노출해요. 그래서 프롬프트에 "존재·행동 장소만, 이동의 에 제외"와 "학생 화면엔 교사용 정보를 숨김"을 꼭 넣으세요.
기본 프롬프트
# 만들 것 한국어 학습용 '에/에서' 빈칸 채우기 학생용 웹앱 # 내용 - 대상: 초급 학습자 - 문항: 10개 ('에' 5개, '에서' 5개) // 존재·행동 장소만. 이동의 에(가다/오다/다니다)는 제외. - 동사: 초급 기본 동사 (있다/없다/먹다/놀다/공부하다 등) - 각 문항마다 정답 해설, 오답 해설 다르게 - 해설은 문법 용어 없이 쉬운 말로 (예: "하는 곳에는 '에서'") // 행동 동사·존재 동사 같은 용어는 학생 화면에 쓰지 않기 # 동작 - '에'/'에서' 버튼을 누르면 즉시 정답/오답 표시 - 해설을 바로 보여주고, 다음 문항으로 넘어가기 - 10문항을 다 풀면 점수 보여주기 - 휴대폰 화면에서도 잘 보이게 # 중요 - 학생 화면에는 문제·선택지·피드백·점수만 보이게 // 정답표, 문항 균형, 제작 설명은 학생에게 보이지 않게
💡 위 프롬프트의 뼈대(만들 것·내용·동작·중요)는 그대로 두고, 목표 문법만 바꾸면 어떤 활동이든 만들 수 있어요. 색이나 버튼 모양 같은 디자인은 굳이 적지 않아도 알아서 만들어줘요.

먼저, 무엇을 고칠지 점검하기

코드를 읽을 필요 없어요. 학생처럼 직접 풀어보며 아래만 확인하세요.

미리보기에서 확인할 것
  • 첫 화면에 문제와 "에/에서" 선택지가 바로 보이나요?
  • 정답을 누르면 바로 정답/오답과 해설이 나오나요?
  • 문항이 10개이고, 마지막에 점수가 나오나요?
  • "가요·와요·다녀요"처럼 이동의 '에'가 섞이지 않았나요?
  • 학생 화면에 정답표·문항 균형·교사용 설명이 안 보이나요?
  • 스마트폰 크기에서도 버튼을 누르기 편한가요?
⚠️ AI는 빠르게 초안을 만드는 조수일 뿐, 교육적 판단은 선생님 몫이에요. 어색한 문장이나 수준에 안 맞는 어휘가 없는지 꼭 확인하세요.
고치는 방법은 두 가지예요

고칠 곳을 찾았으면, 아래 두 방법 중 편한 쪽으로 수정하세요. 대부분은 방법 ①로 충분해요.

방법 ①
말로 고치기 제일 쉬움

고치고 싶은 걸 그냥 말하면 돼요. 코드를 볼 필요도 없어요. 대부분 이걸로 해결돼요.

이렇게 말하면 돼요
3번 문항 문장이 어색해. 더 자연스럽게 바꿔줘. 이동의 '에'가 섞였어. 가다·오다 문장은 빼고 다시 만들어줘. 학생 화면에 정답표가 보여. 문제·선택지·점수만 남겨줘. 버튼이 작아서 휴대폰에서 누르기 힘들어. 크게 해줘. 배경색을 베이지색으로 바꿔줘.
방법 ②
코드로 고치기 더 정밀하게

여러 곳을 한꺼번에 손보거나 세밀하게 조정하고 싶으면, AI한테 "코드를 보여줘"라고 해서 직접 고칠 수 있어요. AI가 만든 문항은 코드 안에 이렇게 들어 있어요. 파란 글씨에 마우스를 올려보세요.

문항 데이터 (일부)
{ sentence: "책이 가방___ 있어요.", answer: "에", correct: "정답이에요. '있어요'는 있는 장소라 '에'.", wrong: "아니에요. 있는 장소를 말하므로 '에'가 맞아요." }, { sentence: "민수는 식당___ 밥을 먹어요.", answer: "에서", correct: "정답이에요. '먹어요'는 행동이라 '에서'.", wrong: "아니에요. 먹는 행동의 장소라 '에서'가 맞아요." }
💡 마우스를 올리면 각 부분이 무슨 뜻인지 알려줘요. 이 부분 문장·정답·해설을 고치면 문항이 바뀌어요.

고치고 → 다시 확인하고를 마음에 들 때까지 반복하면 돼요. 급하면 방법 ①만으로도 충분해요.
4
학생에게 내보내기
링크나 QR코드로 공유해요

앱이 완성되면 링크 하나로 학생에게 줄 수 있어요. 학생은 각자 휴대폰으로 접속해서 자기 속도로 풀면 돼요. 인쇄도 복사도 필요 없어요.

🚀 만든 앱을 인터넷에 올리고 링크·QR코드 만드는 법 → 배포 가이드 보기
🔍 [심화] 문항을 더 꼼꼼히 검토하고 싶다면 — 2단계로 나눠 만들기▶

한 번에 앱까지 만들면 빠르지만, 문항을 하나하나 검토하긴 어려워요. 문항의 질을 꼼꼼히 통제하고 싶으면 두 단계로 나눠서 만드세요.

먼저 문항(JSON)만 만들어 달라고 해서 내용을 검토·수정한 뒤, 그 다음에 앱으로 만드는 방식이에요. AI가 만든 문항을 표로 받아 천천히 확인할 수 있어요.

1단계 — 문항만
// 앱 말고 문항 데이터만 먼저 받기 위 조건으로 문항 10개를 JSON 형식으로만 만들어줘. 각 문항에 문장, 정답, 정답 해설, 오답 해설을 넣어줘.

받은 문항을 검토하고 고친 뒤, "이 데이터로 웹앱 만들어줘"라고 이어서 말하면 돼요. 문항 검토가 왜 중요한지는 프롬프트 가이드에서 더 다뤄요.
