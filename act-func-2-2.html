설계 근거

순서 설명은 "먼저→다음에→마지막으로"라는 시간적 흐름을 따르는 담화예요. 요리 레시피는 이 흐름이 가장 명확하고, 학습자에게 친숙하며, 완성품(음식)이라는 구체적 결과물이 있어서 동기 유발이 쉬워요.

요리는 단계를 건너뛰거나 순서를 바꾸면 결과가 달라지잖아요. "먼저 물을 끓이세요"를 빼먹으면 라면이 안 되거든요. 이 "순서가 중요하다"는 특성이 학습자에게 순서 표현의 필요성을 자연스럽게 인식시켜 줘요.

활동 설계: AI 프롬프트로 자료 만들기

프롬프트 설정

아래 프롬프트를 Claude, ChatGPT, Gemini 등 AI에 입력하면 순서 설명 활동 자료를 JSON 형식으로 생성해요.

# 역할 (Role) 한국어 교육 전문가 # 과제 (Task) TOPIK 2~3급 학습자를 위한 순서 설명 활동 자료 생성 # 조건 (Constraints) - 주제 3개: 라면 끓이기, 도서관에서 책 빌리기, 한국에서 지하철 타기 - 각 주제 4~6단계 - 목표 표현: 먼저, 그다음에, 그리고, 마지막으로, -(으)ㄴ 후에, -기 전에, -(으)면 - 각 단계: 그림 묘사(이미지 생성용) + 설명 문장 + 사용된 표현 # 출력 (Output) JSON 형식

출력 예시

JSON OUTPUT
{
  "topics": [
    {
      "id": 1,
      "title": "라면 끓이기",
      "steps": [
        {
          "order": 1,
          "image_desc": "냄비에 물을 붓는 모습",
          "sentence": "먼저 냄비에 물을 넣으세요.",
          "target_expression": "먼저"
        },
        {
          "order": 2,
          "image_desc": "물이 끓고 있는 냄비",
          "sentence": "물이 끓으면 면과 스프를 넣으세요.",
          "target_expression": "-(으)면"
        },
        {
          "order": 3,
          "image_desc": "끓는 라면을 젓는 모습",
          "sentence": "그다음에 4분 동안 끓이세요.",
          "target_expression": "그다음에"
        },
        {
          "order": 4,
          "image_desc": "라면을 그릇에 담는 모습",
          "sentence": "마지막으로 그릇에 담으세요.",
          "target_expression": "마지막으로"
        }
      ]
    }
  ]
}

JSON 요소 설명

topics → 주제 목록이에요.
steps → 단계별 설명이에요. 순서대로 배열돼요.
image_desc → 이미지 생성용 묘사예요. 각 단계의 그림을 설명해요.
sentence → 설명 문장이에요. 순서 표현이 포함되어 있어요.
target_expression → 목표 표현이에요. 이 단계에서 사용된 순서 표현을 표시해요.

채점 결과

설명 다시 보기

✓ 정답 설명

  1. 먼저 냄비에 물 550ml를 넣으세요.
    시작할 때 사용하는 "먼저"
  2. 물이 끓으면 면과 스프를 넣으세요.
    조건을 나타내는 "-(으)면"
  3. 그다음에 4분 동안 끓이세요.
    다음 단계를 나타내는 "그다음에"
  4. 마지막으로 그릇에 담으세요.
    마지막 단계를 나타내는 "마지막으로"

활동 변형 및 확장

유형 변형 방법 예시
주제 교체 요리 → 다른 순서 설명 은행 계좌 만들기, 비자 신청 절차, 병원 예약하기
난이도 상향 정해진 순서 → 학습자가 배열 섞인 단계 카드를 올바른 순서로 배열하기
상호작용 방식 개인 → 짝 활동 A가 순서를 설명하면 B가 따라 메모하기
창의 활동 주어진 텍스트 → 학습자 제작 자기 나라 음식 레시피를 한국어로 발표

추가 팁

  • 이미지: image_desc를 기반으로 Midjourney, DALL-E 등에 영어로 프롬프트 작성
  • 발음: sentence를 학습자에게 읽어주거나 TTS(Text-to-Speech) 활용
  • 평가: "종이에 그리기", "짝꿍 설명 듣고 평가", "녹음 및 피드백"
