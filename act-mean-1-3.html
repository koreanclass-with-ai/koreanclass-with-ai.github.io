틀린 그림 찾기는 게임적 요소와 언어 연습이 결합된 형태예요. 학습자가 차이를 발견하는 순간 "이걸 어떻게 말하지?"라는 자연스러운 언어 사용 동기가 생기거든요.

두 그림의 차이는 대부분 어떻게 나타날까요? "무엇이 있다/없다", "어디에 있다", "몇 개이다" 같은 존재·위치·수량 표현으로 묘사돼요. 학습자가 일상에서 가장 자주 쓰는 기초 표현을 게임적 맥락 안에서 반복 연습시킬 수 있어요.

BUILD
🤖 Step 1. AI한테 차이점 데이터를 만들어 달라고 하세요

Claude, ChatGPT, Gemini 아무거나 열고 아래 프롬프트를 그대로 붙여넣으세요.

📐 프롬프트를 잘 쓰는 법이 궁금하다면 → 프롬프트 작성 가이드 보기
⚠️ 이 활동에서 특히 주의하세요
이미지 생성 도구로 두 그림을 따로 만들 때, base_description을 공통으로 주고 차이점만 바꾸면 일관된 스타일의 그림 세트가 만들어져요. 예: "현대식 거실, 밝은 조명, 따뜻한 톤" → 이 설명으로 먼저 그림 A를 만들고, 같은 설명으로 차이점을 포함해 그림 B를 만들면 톤이 맞아요.
prompt.txt
# 역할 한국어 교육 전문가 # 과제 TOPIK 1~2급 학습자를 위한 틀린 그림 찾기 활동의 차이점 데이터 생성 # 조건 - 그림 세트: 2개 (거실 장면, 교실 장면) - 각 세트 차이점: 5개 - 차이점 유형: 위치, 존재(있다/없다), 수량, 색상, 크기 변화 포함 - 각 차이점: 원본 상태, 변형 상태, 모범 문장, 사용 표현 포함 - 어휘: 초급~초중급 (TOPIK 1~2급 기본 어휘만) # 출력 JSON 형식
역할은 그대로 두고, 과제와 조건만 바꾸면 다른 주제의 비교 활동도 만들 수 있어요. "교실/카페/부엌" 등 학습자에게 친숙한 장면으로요.
🔁 이렇게 바꿔서 응용하세요

그림 주제를 바꾸거나 난이도를 조절하려면 과제와 조건만 수정하면 돼요.

역할
그대로 두세요. 교육 전문가 역할은 어떤 시각 자료 활동에도 통해요.
과제
차이점을 찾는 대신 "변화 표현으로 설명하기" (-았/었는데 지금은)로 바꿀 수도 있어요.
조건
차이점을 8~10개로 늘리거나 짝 활동으로 변경하세요 (한 명은 그림만 보고 설명, 다른 학습자가 찾기).
출력
앱으로 만들 거면 JSON, 인쇄용이면 "A4 워크시트로 정리해줘"로 바꾸세요.
📤 Step 2. 결과를 확인하세요

AI가 아래처럼 JSON 형식으로 차이점 데이터를 만들어 줘요. 직접 수정해서 쓸 수도 있어요. 파란색 키에 마우스를 올려보세요.

output.json
{ "sets": [ { "id": 1, "scene": "거실", "base_description": "밝은 거실, 소파와 테이블이 있고, 창문으로 햇빛이 들어오는 따뜻한 장면", "differences": [ { "original": "테이블 위에 컵이 있다", "changed": "테이블 위에 컵이 없다", "sentence": "왼쪽 그림에는 컵이 있는데, 오른쪽에는 없어요.", "grammar": "존재 표현 (있다/없다)" }, { "original": "고양이가 소파 위에 있다", "changed": "고양이가 의자 위에 있다", "sentence": "왼쪽에서는 고양이가 소파 위에 있는데, 오른쪽에서는 의자 위에 있어요.", "grammar": "위치 표현 (어디에 있다)" }, { "...": "..." } ] }, { "...": "..." } ] }
scene
→그림의 장면 이름이에요. (거실, 교실 등)
base_description
→이미지 생성 도구에 줄 기본 장면 묘사이에요. 두 그림이 같은 스타일로 만들어지도록 합니다.
original
→그림 A의 원본 상태를 설명해요.
changed
→그림 B의 변형 상태를 설명해요.
sentence
→차이점을 묘사하는 모범 문장이에요. 학습자의 답을 돕는 참고 표현입니다.
⚠️ AI가 만든 차이점 데이터는 꼭 확인하세요. 그림과 실제로 맞는지, 모범 문장이 자연스러운지 검토해야 해요. AI는 빠르게 초안을 만들어 주는 조수고, 교육적 판단은 선생님 몫이에요.
🛠 Step 3. 그림을 만들고 활동 자료를 완성하세요

JSON 데이터가 준비됐으면, 이미지 생성 도구(DALL-E, Midjourney 등)를 사용해요.

Step 1: 그림 A 만들기
base_description + "사진처럼 사실적인 스타일" → 그림 A 생성

Step 2: 그림 B 만들기
base_description + differences의 changed를 모두 포함 + "같은 스타일" → 그림 B 생성

Step 3: 인쇄용 또는 웹 활동으로 배포
완성된 두 그림을 나란히 배치하고, 아래에 "차이점을 찾아보세요" 문제를 출력하거나 웹 인터랙티브로 만들면 돼요.

🎯 완성된 활동의 모습

이 활동은 웹앱 데모 대신 완성된 자료의 모습을 보여줍니다. 학습자들이 풀 때 이런 식으로 진행돼요.

틀린 그림 찾기 · 거실 장면
TOPIK 1~2급 · 두 그림 비교 · 5차이점
그림 A
밝은 거실, 소파에 고양이가 앉아 있고, 테이블 위에는 꽃과 컵이 있어요. 창문으로는 밝은 햇빛이 들어와요.
그림 B
비슷한 거실 장면이지만, 몇 가지가 달라요. 자세히 살펴보고 차이점을 찾아 말해 보세요.
차이점 1
테이블 위의 컵에 대해 뭐가 다를까요?
모범 문장: "왼쪽 그림에는 컵이 있는데, 오른쪽에는 없어요."
문법: 존재 표현 (있다/없다)
차이점 2
고양이의 위치가 어떻게 달라요?
모범 문장: "왼쪽에서는 고양이가 소파 위에 있는데, 오른쪽에서는 의자 위에 있어요."
문법: 위치 표현 (어디에 있다)
차이점 3
꽃의 개수가 어떻게 다를까요?
모범 문장: "왼쪽에는 꽃이 두 송이인데, 오른쪽에는 세 송이예요."
문법: 수량 표현 (몇 개/몇 송이)
학습자는 먼저 두 그림을 보고 자신의 말로 차이를 설명해 보세요. 그 다음 모범 문장을 보면서 "아, 이렇게 말하는구나"라고 배우는 거예요. 틀린 것 찾기보다는 차이점을 어떻게 표현할지를 배우는 활동입니다.
